# Homepage Render Flow

## 1. Entry Route
The homepage route is defined in `routes/tenant.php`:

- `Route::get('/', 'FrontendHomeController@index')->name('frontendHomePage');`

## 2. Middleware and Controller
The request is handled by:
- `app/Http/Controllers/Frontend/FrontendHomeController.php`
- method: `index()`

Controller middleware:
- `maintenanceMode`
- `onlyAppMode`

The controller also checks `Settings('start_site')` and can redirect anonymous users to `login`.

## 3. Homepage Selection Logic
The controller resolves the homepage in this order:
1. Read cached homepage row with key `check_homepage` + `SaasDomain()`.
2. Query `FrontPage::select(['slug', 'is_static'])->where('homepage', 1)->first()`.
3. If the selected homepage is not `/`, dispatch to the route/controller for that slug.
4. If the homepage is `/` and the current theme supports dynamic pages, render the page-builder path.

## 4. Dynamic Homepage Path
`hasDynamicPage()` in `app/Helpers/helper4.php` enables the page-builder path for themes such as `infixlmstheme` and `kidslms`.

When enabled, the controller does:
- `FrontPage::where('slug', '/')->first()`
- `dynamicContentAppend($row->details)`
- `return view('aorapagebuilder::pages.show', compact('row', 'details'));`

That means the homepage structure is stored in the database and post-processed before Blade sees it.

## 5. Content Post-Processing
`dynamicContentAppend()` is a critical helper in `app/Helpers/helper4.php`.
It:
- normalizes legacy absolute URLs
- strips escaped production-domain links where possible
- converts some internal URLs to the current local host
- normalizes `/public/...` paths
- appends dynamic data blocks rendered from `.dynamicData`

## 6. Blade View and Layout
Dynamic homepage view:
- `Modules/AoraPageBuilder/Resources/views/pages/show.blade.php`

This view:
- extends `aorapagebuilder::layouts.master`
- outputs the processed `$details` into the content section
- adds route-specific JS such as `courses.js`

Main layout:
- `Modules/AoraPageBuilder/Resources/views/layouts/master.blade.php`

That layout controls:
- `dir` and RTL classes on `<html>`
- meta tags
- theme CSS/JS loading
- builder/module asset loading
- menu/footer injection
- preloader and global JS state

## 7. Header and Footer Injection
The layout includes:
- `@include(theme('partials._menu'))`
- `@include(theme('partials._footer'))`

`resources/views/frontend/infixlmstheme/partials/_menu.blade.php` chooses the real header partial using `theme('partials.header.'.$header_style)`.
`resources/views/frontend/infixlmstheme/partials/_footer.blade.php` chooses the footer partial using `theme('partials.footer.'.$footer_style)`.

Shared view data for `header_style` and `footer_style` is injected by `app/Providers/AppServiceProvider.php` from `Settings('header_style')` and `Settings('footer_style')`.

A migration also sets the defaults to style `2`:
- `Modules/FrontendManage/Database/Migrations/2024_08_27_171404_AddNewHomePageAndThemeUpdate.php`

So the active header is typically:
- `resources/views/frontend/infixlmstheme/partials/header/2.blade.php`

## 8. Relevant Helper Calls
The homepage chain depends on:
- `currentTheme()`
- `theme()`
- `isRtl()`
- `Settings()`
- `SaasDomain()`
- `dynamicContentAppend()`
- `hasDynamicPage()`
- `getMenuLink()`
- `assetVersion()`

## 9. Dynamic Sections / Builder Mechanics
The homepage is assembled from stored content and builder snippets, not just one controller return.

Important data sources:
- `Modules/FrontendManage\Entities\FrontPage`
- `home_contents` cached in `AppServiceProvider`
- homepage block ordering from `homepage_block_positions`

Important views and admin pages:
- `Modules/FrontendManage/Resources/views/home_content.blade.php`
- `Modules/FrontendManage/Resources/views/front_page/index.blade.php`
- `Modules/FrontendManage/Resources/views/page_content.blade.php`

The homepage HTML itself is often generated by migrations such as:
- `2023_08_30_183652_AddSixHomePageInFrontendPage.php`
- `2024_08_27_171404_AddNewHomePageAndThemeUpdate.php`
- `2024_08_26_174405_AddSecoundPrimaryColor.php`

These migrations render theme snippets into stored HTML, so a DB record can carry the page structure.

## 10. Arabic / RTL Behavior
RTL behavior is controlled by multiple layers:
- `isRtl()` in `app/Helpers/helper3.php`
- `dir="rtl"` and `class="rtl"` in the master layout
- conditional `bootstrap.rtl.min.css`
- conditional `frontend_style_rtl.css`
- header JS that repositions submenus using the document direction
- custom CSS inside the header partial

## 11. Header / Menu / Footer Files
Files that directly affect homepage chrome:
- `resources/views/frontend/infixlmstheme/partials/_menu.blade.php`
- `resources/views/frontend/infixlmstheme/partials/header/2.blade.php`
- `resources/views/frontend/infixlmstheme/partials/_footer.blade.php`
- `Modules/AoraPageBuilder/Resources/views/layouts/master.blade.php`
- `app/Providers/AppServiceProvider.php`
- `app/Helpers/helper3.php`
- `app/Helpers/helper4.php`

## 12. Why the Homepage Can Look Broken Locally
Typical causes in this architecture:
- stale `front_pages.details` HTML containing production URLs
- wrong `header_style` or `footer_style`
- RTL CSS missing or loaded in the wrong order
- module assets not reachable through the local asset proxy route
- cached config/view output
- theme branch mismatch

Debugging should always trace the page from route -> controller -> stored HTML -> final CSS/JS load order.
